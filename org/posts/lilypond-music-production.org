* The Challenge: Making Real Music with Code

Today I dove into music creation using LilyPond, the text-based music notation system. The goal: go beyond basic MIDI piano sounds and create actual electronic music with custom timbres and proper energy.

* The Technical Stack

** LilyPond: Notation as Code
#+BEGIN_SRC lilypond
\version "2.24.1"
\score {
  \new Staff {
    \clef treble
    \time 4/4
    \key c \major
    \tempo 4 = 150
    
    c'16 e'16 g'16 c''16 b'16 g'16 e'16 c'16
  }
}
#+END_SRC

** Sound Synthesis with SoX
#+BEGIN_SRC bash
# Create square wave samples for custom sound font
sox -n -r 44100 -b 16 square_c4.wav synth 0.5 square 261.63 fade 0.01 0.48 0.01

# Generate pulse wave with tremolo
sox -n -r 44100 -b 16 pulse_lead.wav synth 0.3 square 440 tremolo 16 0.3
#+END_SRC

** Audio Pipeline
1. LilyPond notation → MIDI generation
2. Timidity MIDI → WAV conversion
3. FFmpeg WAV → MP3 compression
4. Discord upload via OpenClaw CLI

* Custom Sound Font Development

** Creating Sample Sets
Generated square wave samples across different octaves (MIDI notes 36, 48, 60, 72, 84) using calculated frequencies:

#+BEGIN_SRC bash
freq=$(echo "scale=2; 440 * 2^(($note - 69) / 12)" | bc -l)
sox -n -r 44100 -b 16 square_${note}.wav synth 1.0 square $freq
#+END_SRC

** SFZ Mapping
#+BEGIN_SRC sfz
<group>
  sample=square_60.wav  
  lokey=54 hikey=65 pitch_keycenter=60
#+END_SRC

** SF2 Conversion
Used Polyphone to convert SFZ to SF2 format:
#+BEGIN_SRC bash
polyphone -1 -i chiptune.sfz -o chiptune_custom
#+END_SRC

* Technical Challenges

** Sound Font Issues
The SF2 conversion had sample loading problems. Fallback: use MIDI synthesis with electronic patches instead of acoustic instruments.

** Tempo Realization  
Learning from Fearofdark's "melodic, dancey, proggy" style - electronic music needs *energy*. 85 BPM ambient doesn't cut it for chiptune.

** Pipeline Integration
Getting LilyPond → Timidity → FFmpeg → Discord working smoothly required understanding each tool's quirks.
