* Technical Deep-Dive: AST-Based Validation

The key insight was using **AST parsing instead of execution**. Here's why:

#+BEGIN_SRC python
def assert_for_syntax_parses(self, source: str):
    """Test that for statement syntax parses correctly."""
    try:
        tree = ast.parse(source, mode='exec')
        # Verify the AST contains expected node types
        for node in ast.walk(tree):
            if isinstance(node, ast.For):
                return tree
        pytest.fail(f"Expected For node not found")
    except SyntaxError as e:
        pytest.fail(f"For syntax failed to parse: {e}")
#+END_SRC

This approach gives us:

1. **Cross-implementation compatibility** - AST structure is standardized
2. **Version awareness** - Can test syntax without runtime dependencies
3. **Security** - No code execution, just parsing validation
4. **Precision** - Tests exactly what the Language Reference specifies

* The Pattern Matching Challenge

Section 8.10 (Match Statements) was particularly interesting because it's **Python 3.10+ only**. How do you test new syntax on older implementations?

#+BEGIN_SRC python
@pytest.mark.min_version_3_10
def test_match_literal_patterns(self, tester):
    """Test literal patterns in match statements"""
    match_statements = [
        """match value:
    case 42:
        return "found answer"
    case "hello":
        return "greeting" """,
    ]
    
    for stmt in match_statements:
        if sys.version_info < (3, 10):
            pytest.skip("Match statements require Python 3.10+")
        tree = tester.assert_match_syntax_parses(stmt)
#+END_SRC

The solution: **graceful degradation with pytest markers**. Tests run on compatible versions, skip cleanly on older ones.

* Technical Elegance Through Constraints

The Language Reference's formal grammar created natural constraints that led to elegant solutions:

#+BEGIN_SRC python
# Grammar: for_stmt: 'for' exprlist 'in' testlist ':' suite ['else' ':' suite]

def test_for_with_else_clauses(self, tester):
    """Test for...else syntax compliance"""
    for_else_statements = [
        """for i in range(3):
    print(i)
else:
    print("done")""",
    ]
    
    for stmt in for_else_statements:
        tree = tester.assert_for_syntax_parses(stmt)
        for_nodes = tester.get_for_nodes(stmt)
        assert len(for_nodes[0].orelse) > 0  # Else clause exists
#+END_SRC

The grammar **tells you exactly what to test**. No guessing about edge cases or coverage gaps.
